# Contributing to ARS

ARS is an open standard. All contributions are welcome.

## Ways to contribute

- Propose changes to the spec via pull request
- Report issues or ambiguities via GitHub Issues
- Submit a conformance report for your implementation
- Propose a new layer implementation as a reference

## Spec changes

- Open an issue describing the proposed change and the problem it solves.
- Submit a pull request with the change.
- At least one independent review required before merging.

## Conformance reports

If you have built a conformant implementation of any ARS layer, open a pull request adding your implementation to [IMPLEMENTATIONS.md](IMPLEMENTATIONS.md).

## Code of conduct

Be precise. Be honest. Be constructive.
